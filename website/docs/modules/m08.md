# Module 08: OpenAI ES for RL — Evolution at Scale

In 2017, OpenAI showed that a simple ES could compete with sophisticated RL algorithms on Atari and MuJoCo by throwing thousands of CPUs at the problem. This module recreates that result and explores why ES is so naturally parallelizable — and where it still falls short compared to policy gradients.

## Learning Objectives

1. Implement the OpenAI ES algorithm (Gaussian parameter-space exploration with shared noise table).
2. Train an ES agent on CartPole and a continuous control task.
3. Understand the communication efficiency of ES (just send scalars, not gradients).
4. Compare wall-clock time vs. sample efficiency between ES and PPO.
5. Implement basic parallelism using Python multiprocessing.
